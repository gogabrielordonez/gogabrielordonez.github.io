GenAI Semantic Search

Full-Stack Vector Database Application with Streamlit

Role: Full-Stack AI Engineer Duration: 2024 Type: End-to-End Application
View on GitHub

The Challenge

Traditional keyword-based search fails when users don't know exact terminology. Enterprise systems needed a way to find documents by meaning, not just matching words.

Additionally, teams needed flexibility to choose between local development (FAISS) and production-scale cloud solutions (Pinecone) without rewriting application code.

  • Keyword search misses semantically similar content
  • No unified interface for different vector databases
  • High API costs for embedding generation
  • Complex setup for non-technical stakeholders

The Solution

Built a full-stack semantic search application with a clean abstraction layer that supports multiple vector stores and embedding providers.

The solution includes an interactive Streamlit dashboard for document indexing, semantic search, and real-time analyticsâ€”making AI-powered search accessible to all users.

  • Vector store abstraction for FAISS and Pinecone
  • Embedding service with intelligent caching
  • Streamlit UI for intuitive interaction
  • Real-time performance analytics

Implementation Highlights

Vector Store Abstraction

Clean interface enabling seamless switching between FAISS and Pinecone:

class VectorStoreBase(ABC):
    """Abstract base class for vector store implementations"""

    @abstractmethod
    def add_documents(self, documents: List[Document]) -> int:
        """Add documents to the vector store"""
        pass

    @abstractmethod
    def search(self, query_embedding: List[float], top_k: int = 5) -> List[SearchResult]:
        """Search for similar documents"""
        pass

class VectorStoreFactory:
    @staticmethod
    def create(provider: str = "faiss", dimension: int = 1536, **kwargs) -> VectorStoreBase:
        if provider.lower() == "faiss":
            return FAISSVectorStore(dimension=dimension, **kwargs)
        elif provider.lower() == "pinecone":
            return PineconeVectorStore(dimension=dimension, **kwargs)
        else:
            raise ValueError(f"Unknown provider: {provider}")

Embedding Service with Caching

Intelligent caching reduces API costs by avoiding redundant embedding calls:

class EmbeddingService:
    def __init__(self, provider: str = "mock", cache_enabled: bool = True):
        self.cache_enabled = cache_enabled
        self.cache: Dict[str, List[float]] = {}
        self.stats = {"hits": 0, "misses": 0, "total_tokens": 0}

    def embed(self, text: str) -> List[float]:
        cache_key = hashlib.md5(text.encode()).hexdigest()

        if self.cache_enabled and cache_key in self.cache:
            self.stats["hits"] += 1
            return self.cache[cache_key]

        self.stats["misses"] += 1
        result = self.provider.embed(text)

        if self.cache_enabled:
            self.cache[cache_key] = result.embedding

        return result.embedding

Semantic Search Pipeline

End-to-end search with timing metrics:

def search(self, query: str, top_k: int = 5, min_score: float = 0.0) -> SearchResponse:
    # Generate query embedding
    embed_start = time.perf_counter()
    query_embedding = self.embedding_service.embed(query)
    embed_time = (time.perf_counter() - embed_start) * 1000

    # Search vector store
    search_start = time.perf_counter()
    results = self.vector_store.search(query_embedding, top_k=top_k)
    search_time = (time.perf_counter() - search_start) * 1000

    # Filter by minimum score
    if min_score > 0:
        results = [r for r in results if r.score >= min_score]

    return SearchResponse(
        query=query,
        results=results,
        total_results=len(results),
        search_time_ms=round(search_time, 2),
        embedding_time_ms=round(embed_time, 2)
    )

Technical Decisions

Why Abstract Factory Pattern for Vector Stores?

The factory pattern allows runtime selection of vector store providers. Development uses FAISS (free, local, fast iteration), while production deploys to Pinecone (managed, scalable, reliable). Same codebase, different configurations.

Why Multiple Embedding Providers?

OpenAI embeddings offer best-in-class quality but have API costs. Sentence Transformers run locally for privacy-sensitive use cases. Mock embeddings enable deterministic testing without network calls.

Why Streamlit for the Frontend?

Streamlit enables rapid prototyping with pure Pythonâ€”no JavaScript required. Perfect for data-centric applications where the focus is on functionality over complex UI. Built-in state management and caching complement the backend architecture.

Why L2 Distance for FAISS?

L2 (Euclidean) distance works well with normalized embeddings and provides intuitive similarity scores. For production with Pinecone, cosine similarity is used as it's optimized for their infrastructure.

Results & Impact

Production-Ready Architecture

Clean separation of concerns enables independent scaling of embedding generation, vector storage, and search logic. The abstraction layer means infrastructure changes don't require application rewrites.

Developer Experience

Simple API with sensible defaults. Index a document in one line: engine.index_document("Your text here"). Search with natural language: engine.search("machine learning basics").

Cost Optimization

Embedding cache eliminates redundant API calls. FAISS option provides zero-cost development. Batch processing reduces per-document overhead for large imports.
